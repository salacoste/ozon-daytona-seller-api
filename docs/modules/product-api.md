# ProductAPI - Управление каталогом товаров

## 🎯 Что это такое

ProductAPI - это центральный модуль для управления всем жизненным циклом товаров в каталоге Ozon: от создания карточек до анализа их эффективности.

## 💼 Бизнес-задачи которые решает

- **Управление ассортиментом** - полный контроль над каталогом товаров
- **Оптимизация контента** - улучшение описаний и характеристик
- **Контроль видимости** - мониторинг статусов модерации
- **Массовые операции** - эффективная работа с большими каталогами

---

## 📋 Основные функции

### 1. **Создание и редактирование товаров**
**Что делает:** Добавление новых товаров и обновление существующих

**Методы:**
- `createProduct()` - создать новую карточку товара
- `updateProduct()` - обновить информацию о товаре
- `importProducts()` - массовый импорт товаров
- `uploadImages()` - загрузить фотографии товаров

**Бизнес-ценность:** 
- Быстрое пополнение каталога
- Поддержание актуальности информации
- Экономия времени на рутинных операциях

---

### 2. **Просмотр и поиск товаров**
**Что делает:** Получение информации о товарах в каталоге

**Методы:**
- `listProducts()` - получить список всех товаров
- `getProductInfo()` - детальная информация о товаре
- `searchProducts()` - поиск товаров по критериям
- `getProductsByCategory()` - товары в определённой категории

**Бизнес-ценность:**
- Контроль над ассортиментом
- Анализ структуры каталога
- Быстрый поиск нужных товаров

---

### 3. **Управление характеристиками**
**Что делает:** Работа с атрибутами и свойствами товаров

**Методы:**
- `getProductAttributes()` - получить характеристики товара
- `updateAttributes()` - обновить атрибуты
- `getRequiredAttributes()` - обязательные характеристики для категории
- `validateAttributes()` - проверить корректность атрибутов

**Бизнес-ценность:**
- Соответствие требованиям категории
- Улучшение ранжирования в поиске
- Повышение конверсии покупок

---

### 4. **Контроль статусов модерации**
**Что делает:** Отслеживание прохождения товаров через модерацию

**Методы:**
- `getModerationStatus()` - статус модерации товара
- `getModerationErrors()` - ошибки модерации
- `resubmitForModeration()` - повторная подача на модерацию
- `getBlockedProducts()` - список заблокированных товаров

**Бизнес-ценность:**
- Быстрое исправление ошибок модерации
- Максимизация видимых товаров
- Соблюдение правил Ozon

---

### 5. **Архивирование и удаление**
**Что делает:** Управление жизненным циклом товаров

**Методы:**
- `archiveProduct()` - архивировать товар
- `unarchiveProduct()` - восстановить из архива
- `deleteProduct()` - удалить товар
- `getArchivedProducts()` - список архивированных товаров

**Бизнес-ценность:**
- Оптимизация каталога
- Удаление неактуальных позиций
- Сезонное управление ассортиментом

---

### 6. **Аналитика товаров**
**Что делает:** Статистика эффективности товаров

**Методы:**
- `getProductPerformance()` - показатели эффективности товара
- `getViewsStatistics()` - статистика просмотров
- `getConversionRate()` - конверсия товара
- `getTopPerformingProducts()` - топ товаров по продажам

**Бизнес-ценность:**
- Выявление популярных товаров
- Оптимизация ассортимента
- Принятие решений о закупках

---

## 📊 Ключевые метрики

- **Статус модерации** - доля прошедших модерацию товаров
- **Видимость в поиске** - позиции товаров в результатах поиска
- **Конверсия товаров** - процент просмотров, приводящих к покупке
- **Качество контента** - полнота заполнения характеристик

## 💡 Типовые сценарии использования

### Сценарий 1: Добавление нового товара
1. Определить подходящую категорию
2. Заполнить обязательные характеристики
3. Загрузить качественные фотографии
4. Написать продающее описание
5. Отправить на модерацию
6. Отслеживать статус модерации

### Сценарий 2: Массовое обновление каталога
1. Экспортировать текущий каталог
2. Подготовить обновлённые данные
3. Выполнить массовый импорт изменений
4. Проверить результаты обновления
5. Исправить ошибки при необходимости

### Сценарий 3: Оптимизация товаров по эффективности
1. Получить статистику по всем товарам
2. Выявить низкоэффективные позиции
3. Проанализировать причины низкой эффективности
4. Обновить описания и характеристики
5. Отследить изменения в показателях

### Сценарий 4: Сезонное управление ассортиментом
1. Архивировать товары прошлого сезона
2. Восстановить актуальные для сезона позиции
3. Добавить новые сезонные товары
4. Обновить цены согласно сезонности

---

## ⚠️ Важные особенности

- **Модерация** - все товары проходят проверку службой модерации Ozon
- **Обязательные атрибуты** - для каждой категории есть обязательные характеристики
- **Качество фото** - требования к размеру и качеству изображений
- **Уникальность** - товары должны быть уникальными в рамках категории

## 🎨 Советы по оптимизации товаров

### Название товара
- Используйте ключевые слова для поиска
- Укажите основные характеристики
- Длина: 50-150 символов

### Описание товара  
- Подробно опишите преимущества
- Используйте структурированный текст
- Включите размеры и материалы

### Фотографии
- Минимум 5 фото разных ракурсов
- Разрешение не менее 900x900 пикселей
- Белый или нейтральный фон

### Характеристики
- Заполните все доступные атрибуты
- Используйте точные значения
- Добавьте дополнительные свойства
